# Yobou Market — Google Play Store launch guide

This is the step-by-step recipe for taking the Capacitor-wrapped Yobou app from
`./APP_shopper_and_buyer/android/app/build/outputs/bundle/release/app-release.aab` to a live
Google Play listing.

The Capacitor build is already wired up. This document covers what only the
publisher (you) can do outside the repo: Play Console account, store listing
assets, content rating, and the actual upload.

---

## 1. What is already prepared in this repo

| Asset | Path |
|---|---|
| Signed Android App Bundle (AAB) | `APP_shopper_and_buyer/android/app/build/outputs/bundle/release/app-release.aab` |
| Signed APK (debug-ish, for sideload testing) | `APP_shopper_and_buyer/android/app/build/outputs/apk/release/app-release.apk` |

| Keystore credentials (gradle.properties) | `APP_shopper_and_buyer/android/gradle.properties` (`YOBOU_*`) |
| App ID | `com.yobou.market` |
| App name | Yobou Market |
| Splash + status bar | Brand blue `#0034b9` |
| Network security config | TLS-only (`/api/*` must be HTTPS) |
| Privacy policy page | `server/public/privacy.html` → served at `/privacy` |

**Back up the keystore now.** Losing `yobou-release.keystore` permanently
disables updates on every device that installed the original. Store a copy in
1Password / Bitwarden / an offline USB drive.

## 2. One-time Google Play Console setup

1. Create a Google Play Developer account: <https://play.google.com/console/signup>
   ($25 one-time fee). Account verification takes 24–48 h.
2. In the console: **Create app** → name "Yobou Market", default language
   English, app type "App" (not Game), free.
3. **Dashboard → Set up your app** — fill in:
   - App access: "All functionality is available without special access" unless you add a real signup gate later.
   - Ads: "No, my app does not contain ads".
   - Content rating: complete the questionnaire (IARC). E-commerce without user-generated content, no gambling, no violence → ESRB "Everyone" / PEGI "3".
   - Target audience: not children.
   - Government apps: No.
   - Financial features: select "App supports transactions" (it does — simulated payments — still tick this so Play knows it's an e-commerce app).
   - Data safety: declare the categories in §3.
   - Health apps: No.

## 3. Data safety form (Play Console → App content → Data safety)

Declare the data the seeded app handles. You can paste this into the form's
"Notes" field for the reviewer:

| Data category | Collected? | Shared? | Purpose | User controls |
|---|---|---|---|---|
| Account info — name, email | Yes | No | Account creation, orders | Delete account on request |
| Passwords | Yes (hashed) | No | Authentication | Reset via email |
| Purchase history | Yes | No | Order tracking, refunds | Delete account on request |
| App activity (page views, actions) | No (no analytics SDK) | — | — | — |
| Device ID / advertising ID | No | — | — | — |
| Location | No | — | — | — |

Mention that **payments are simulated** and no real card data is collected or
transmitted to a processor.

## 6. API host before you upload

The Android app talks to whatever URL is baked into the web bundle via
`VITE_API_BASE`. For the **first release** you must:

1. Deploy the API to a public HTTPS host. Recommended one-click options:
   - **Render** — connect the repo, set `DATABASE_URL` to a Postgres
     connection string, set the JWT secrets, set `NODE_ENV=production`.
     Render gives you a `*.onrender.com` URL with HTTPS for free.
   - **Railway** — similar; one-click Postgres add-on.
   - **Fly.io** — `fly launch`, `fly postgres create`, deploy.
2. Set `CORS_ORIGIN` to include the deploy origin (the Android app itself
   is not subject to CORS, but the React app fetches the API directly
   from the device using Capacitor's `https://localhost` scheme — the
   server must allow the Android WebView origin. The current
   `cors({ origin: CORS_ORIGIN })` config in `server/src/index.js`
   trusts whatever you list in `CORS_ORIGIN`; add the production host
   there).
3. Rebuild the web bundle with the new API URL baked in:

   ```bash
   # from repo root
   cd client
   VITE_API_BASE=https://api.yobou.example npm run build
   npx cap sync android
   cd android
   .\gradlew.bat bundleRelease    # rebuilds AAB
   ```

   The resulting `app/build/outputs/bundle/release/app-release.aab` is
   what you upload.

If you ship the demo app without changing `VITE_API_BASE`, it will try to
hit `http://localhost:4000`, which only works on a developer machine.
**Always rebuild before upload.**

## 7. Upload to Play Console

1. Play Console → your app → **Release → Production → Create new release**.
2. Upload `app-release.aab`.
3. Add release name (e.g. `1.0.0`) and release notes (what's new).
4. Review the AAB's pre-launch report (Play runs the APK on real devices
   and reports crashes). Address any Class A issues.
5. Roll out: start with **Internal testing** → **Closed testing** → **Production**.
   Internal testing is the safest first step — only people you add by email
   can install it, and you skip the full review queue for that track.

---

## 8. Internal testing fast-track (recommended first release)

1. **Testing → Internal testing → Create new release**.
2. Upload the same AAB.
3. Create an email list of testers (your own Gmail works).
4. Testers click the opt-in link, install from Play Store, and report.
5. Once stable, promote to **Closed testing** (add a wider list) then to
   **Production**.

Internal-testing reviews still happen but skip several production gates,
so first-time apps get approved faster there.

## 9. Post-launch checklist

- [ ] Confirm `/privacy` resolves over HTTPS and Play Console accepts it
- [ ] Re-upload a new AAB whenever you bump `versionCode` / `versionName`
- [ ] Back up `yobou-release.keystore` in at least two secure locations
- [ ] Monitor **Vitals → Crashes & ANRs** in Play Console
- [ ] When real payments are wired up, switch the data safety form to
      declare the actual processor (Stripe, Adyen, etc.) and add the
      Payments policy URL Play requires

## 10. Troubleshooting

**AAB rejected: "App Bundle was not signed"**
→ You're uploading a debug build. Confirm the bundle path includes
`outputs/bundle/release/` and not `outputs/apk/debug/`.

**Lint fails with `Lint found fatal errors`**
→ Run `cd APP_shopper_and_buyer/android && .\gradlew.bat lintRelease` to see the
specific error. Common: empty `<domain-config>` (removed in this repo),
missing launcher icon density, or a wrong `versionCode` format.

**AAB rejected: `minSdkVersion 24`**
→ Play Store accepts 24+. The default `minSdkVersion = 24` in
`APP_shopper_and_buyer/android/variables.gradle` is fine.

**App opens to a blank white screen**
→ The bundled web assets couldn't load. Either `npx cap sync` wasn't run
after `npm run build`, or `VITE_API_BASE` points somewhere unreachable.
Run `adb logcat | grep -i chromium` to see the real error.

**`CORS_ORIGIN` blocking requests from the app**
→ The WebView origin is `https://localhost`. Set `CORS_ORIGIN` to a
comma-separated list of your public host(s) plus `https://localhost` if
your server logic checks the Origin header.
